# Blog Feature Delivery

当任务是为博客项目实现一个新功能时，按下面顺序推进：

1. 先明确用户故事、影响页面、影响 API、数据结构和回归范围。
2. 先读代码，再决定拆分前端、后端、测试和 CR 子任务。
3. 前端关注交互状态、空数据、错误态和移动端。
4. 后端关注 schema、鉴权、幂等性和兼容旧字段。
5. 测试至少覆盖：
   - 正常流程
   - 参数非法
   - 权限不足
   - 回归路径
6. 如果需要对外说明，补一份变更文档和流程图。

默认输出结构：

- 目标
- 现状
- 变更方案
- 实施步骤
- 验证方式
- 风险和回退
